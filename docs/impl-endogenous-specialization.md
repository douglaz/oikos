# Implementation Spec: the endogenous specialization economy

## Purpose

Make specialization **emerge from a subsistence base and sustain on market trade
alone** — no curated capital/food/input advances. This is the milestone the
twelve money-circulation experiments converged on (`docs/experiment-money-
circulation.md`): a stone-age colony (subsistence, Experiment 10) grows a
grain→flour→bread division of labor that **self-organizes** and keeps running,
because producers acquire their inputs through the **real market** rather than
having them hand-placed.

It is the falsification of the Experiment-12 "scaffolded" verdict: the only thing
that has sustained the chain so far is `run_input_advance` *placing* inputs in
producers' hands each tick (a planner). Strip every curated phase and the chain
dies ~tick 150 (Experiments 12 ablations, 13). This milestone must make it live
**without** them.

It is NOT firms/wage-labor (rejected as an overclaim — Codex review), NOT
banks/fiat/credit (later eras), NOT value-scale surgery (do not reorder a
producer's wants), and NOT a change to econ's *market-clearing* behavior (the six
econ conformance goldens stay byte-identical — any econ edit is additive and
gated).

## Verified Base Facts (2026-06-17, oikos @ `4f5fb57`)

1. **The subsistence baseline works.** `frontier_in_kind` (Experiment 10) keeps a
   colony fed (hunger ~2.7) and conserved for 1600 ticks — but by *redistributing*
   subsistence, not by reviving the market chain. The right stone-age baseline.
2. **The core blocker (Experiments 9, 13): producers do not buy inputs in the
   market.** Adoption uses the ordinal **project-bundle** appraisal
   (`recipe_adoption_pays_for_money`, `sim/src/settlement.rs:7572` →
   `appraise_project_bundle_for_money`, `econ/src/bundle.rs:50`). But the input
   **bid** is generated by the *generic* spot path (`ensure_bid`,
   `econ/src/society.rs:2408` → `Agent::reservation_bid_for_money`,
   `econ/src/agent.rs:356`), which is **blind to the recipe** — it sees "buy
   grain," not "buy grain to complete milling that provisions my end." The
   producer's input want is inserted *below* its consumption/savings wants
   (`producer_scale_extension` / `scale_input_insert_position`,
   `sim/src/settlement.rs:7746,7761`), so `allocated_money_before_rank` protects
   its money for the savings want and `reservation_bid_for_money(grain)` returns
   `None`. Live order-book trace: gatherers ASK grain, **no one BIDS**.
3. **The faithful direction (Codex): project-aware bids in the REAL order book.**
   The bid must reuse the bundle appraisal (an imputed reservation = expected
   output value − costs, NOT a scalar `recipe_is_profitable` on a stale
   `realized_price`, and NOT moving the input want's rank). The order book is
   `Society::books` (`econ/src/society.rs:245`).
4. **The sim-phase attempt (Experiment 13) is insufficient.**
   `run_project_input_bids` (`sim/src/settlement.rs:5459`) has a producer buy one
   input unit at an imputed price from a willing seller — conserved, endogenous —
   but the chain still collapses ~t150 (bread/800 = 666 vs the scaffold's 7011).
   Causes diagnosed: (a) the revolving loan's repayment
   (`run_capital_repayment`, `sim/src/settlement.rs:5550`) sweeps the producer
   cash-light before it can re-buy; (b) the cold-start (a miller needs a realized
   flour price to adopt, but no flour is made until it adopts); (c) the
   grain→flour→bread pipeline lag. A single per-tick sim phase does not resolve
   these.
5. **`recurring_motive` keeps producers adopted while profitable**
   (`run_role_choice`, `sim/src/settlement.rs:5702`) — the satiation/retirement
   fix; it is sound and stays. **Caveat (Codex):** it currently reads
   `recipe_is_profitable` on the *last* realized price (can go stale) and treats a
   missing input price as zero cost — tighten or fold into the bundle appraisal.
6. **Conservation/determinism** inherited: `EconTickReport::conserves()` (the
   `after = before + sources − sinks` identity). Every new flow is a conserved
   transfer or an accounted sink. Integer state, no loop-time RNG, `BTreeMap`/`Vec`.
7. **Goldens byte-identical by construction**: all of this is game-only (the lab
   has no chain colony); any econ edit (the order-book bid hook) is additive and
   gated, so the six conformance goldens are unchanged.

## Milestone Boundary

Includes:

- **Project-aware input bids posted into the real order book.** A new
  `project_input_reservation_for_money(agent, recipe, output_price, operating_cost,
  money_good) -> Option<Gold>` that reuses the **bundle appraisal** (not the scalar
  heuristic) to compute the max the producer will pay for its input as an imputed
  residual. The producer posts this as a **normal market bid** (it can fill
  against a willing seller's ask; if none crosses, no input — no placement). The
  generic low-ranked input want for that good is **suppressed** to avoid a
  duplicate bid. (Codex spec #4; this likely needs a small additive, gated econ
  hook to inject/price a producer's spot bid — goldens unaffected.)
- **Working-capital persistence across ticks.** Resolve the loan/repayment sweep
  so a producer retains enough money to keep bidding: the loan funds a persistent
  working-capital float, repaid only from realized **sales surplus** (not swept to
  zero each tick), OR the producer simply retains its earnings (recurring_motive
  already prevents the satiation that the sweep was guarding against). No
  fiduciary credit — real money only.
- **Cold-start bootstrap** so the first flour/bread realized price exists to drive
  adoption (seeded `latent_flour_seed` / `bread_buffer` already exist; define the
  bootstrap explicitly so the pipeline fills deterministically).
- **A scaling / churning producer pool** so output tracks population: a hungry
  colonist can take up an unserved trade (replacement / re-entry), and the latent
  pool is not a fixed handful — this is what keeps hunger bounded as population
  grows (the Experiment-12 caveat).
- The **subsistence base** (Experiment 10) underneath, so no one starves during
  the transition.
- `recurring_motive` (tightened per fact 5).
- An `endogenous` DoD config with **every curated advance OFF**
  (`capital_advance` only as a bootstrap float if needed, `subsistence_advance`,
  `input_advance` all OFF).
- Conservation across every new flow; determinism; viewer surfacing of
  market-sourced input volume vs hunger; acceptance tests; README + `engine-
  divergence.md`.

Excludes:

- curated food/input PLACEMENT (`subsistence_advance`, `input_advance`) in the DoD
  config — those are the scaffold being replaced (keep the code/flags for
  comparison, but the endogenous DoD must pass with them OFF);
- firms / wage-labor; banks / fiat / credit (later eras);
- value-scale surgery (reordering the producer's wants);
- any change to econ market-CLEARING behavior — six goldens byte-identical;
- no `HashMap` in logic; magnitudes SIGN/direction only (input.input > 0 vs 0;
  bread sustained vs collapses; hunger bounded vs drifting) plus exact conservation.

## Domain Semantics

### Endogenous vs scaffolded (the whole point)

A producer must obtain its input by **paying for it in the market** (its own
money, a willing seller, a price it rationally accepts), not by a planner placing
it. The single clean metric (Codex): **tail market-sourced input units per tick,
with all curated transfer phases OFF.** If grain/flour reach producers only via a
transfer phase, it is scaffolded; if via order-book trades, it is endogenous.

### Imputed bid (Menger)

The value of a higher-order good (grain to a miller) is **imputed** from the final
good it helps produce: the miller's reservation for grain is the realized value of
the flour it will yield, minus operating cost (a residual), bounded so the project
still nets a gain — exactly the bundle appraisal that already justifies *adoption*.
The fix is to make the *bid* use that same appraisal, so an adopted producer with
working capital actually bids.

### Why the pieces are interdependent

The bid is useless without working capital to fund it; working capital is pointless
if the bid is project-blind; both need the cold-start to produce the first
realized prices; and the whole thing only stays fed (and only scales) with the
subsistence base and a churning producer pool. The milestone must compose all of
them — the acceptance test is the composition.

## Acceptance Tests

`sim/tests/endogenous_economy.rs` (+ unit tests):

1. `endogenous_run_is_deterministic` — same `(seed, config)` → byte-identical.
2. `inputs_acquired_by_market_not_placement` — with every curated transfer phase
   OFF, `grain.input > 0` AND `flour.input > 0` **after tick 300**, and the input
   units are sourced by order-book trades (a producer's gold falls and a seller's
   rises for the input), not by a transfer phase. THE clean endogenous metric.
3. `specialization_sustains` — `bread.made > 0` through tick 800 (and 1600); the
   chain does not collapse to the ~t150 wall.
4. `hunger_stays_bounded` — colony hunger stays below a fixed ceiling and does not
   monotonically drift toward crisis (the stable-economy vs managed-decline
   distinction; pair with a population-trend read).
5. `producers_adopted_or_replaced` — producers stay adopted or the pool churns
   (a new colonist takes up the trade) — output tracks population.
6. `endogenous_conserves` — whole-system conservation holds every tick across all
   new flows (bids, trades, any working-capital float).
7. `econ_unchanged` — full workspace suite; six econ goldens byte-identical; all
   prior tests green; `cargo clippy --workspace --all-targets -- -D warnings`;
   `cargo fmt --check`.

## Handoff Notes

- The bid MUST be a real order-book order priced by the **bundle appraisal**, not
  a sim-phase placement and not the scalar `recipe_is_profitable`. Suppress the
  generic input want to avoid duplicate bids. The single clean metric (test 2)
  is the tripwire for "endogenous, not scaffolded."
- Working-capital persistence is the second load-bearing piece: do not sweep the
  producer cash-light before it can bid. `recurring_motive` already removes the
  satiation that the sweep guarded, so the producer may retain earnings.
- Cold-start: the first flour/bread realized price must exist (seeded buffers);
  confirm the pipeline fills deterministically so millers→bakers adopt in order.
- Keep the curated-advance flags/scenarios for COMPARISON, but the endogenous DoD
  config must pass with them OFF. If a test only passes with a transfer phase on,
  it is still scaffolded — fix the market path, don't re-enable placement.
- Six econ goldens byte-identical; any econ order-book hook is additive/gated.
  Conserved every tick; determinism; SIGN-only magnitudes.
- `git add` new files; gitignore stray build artifacts.
